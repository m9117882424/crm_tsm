Clean server deployment plan

Goal: deploy Leantime CRM from zero on a stronger server.

Current production URL:
https://crm.equippulse.com

Reason for clean deployment:
There is no important CRM data yet. Projects, tasks and users can be recreated manually.

What will be migrated:
- crm_tsm GitHub repository;
- deployment rules;
- Docker Compose configuration;
- domain name crm.equippulse.com.

What will not be migrated:
- old Leantime database;
- old uploaded files;
- old Docker volumes;
- old test users;
- old test tasks.

Clean deployment approach:
1. Keep old server running until the new server is verified.
2. Install Docker, Nginx and Certbot on the new server.
3. Clone crm_tsm repository to /opt/crm_tsm.
4. Create a fresh .env file with new database passwords.
5. Start Leantime with Docker Compose.
6. Configure Nginx reverse proxy to 127.0.0.1:8082.
7. Change DNS A record for crm.equippulse.com to the new server IP.
8. Issue SSL certificate on the new server.
9. Complete Leantime installation wizard.
10. Create administrator, users, projects and roles from zero.
11. Verify web access, login, files, email and notifications.
12. Only then stop Leantime on the old server.

Do not delete the old server before successful verification.
