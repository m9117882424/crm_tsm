# Типы задач и статусы

## Типы задач

| Тип | Когда использовать |
|---|---|
| Монтаж | Установка оборудования |
| Демонтаж | Снятие оборудования |
| Проверка | Диагностика, тестирование |
| Дефект | Неисправность, ошибка, инцидент |
| Заявка | Входящий запрос от отдела/пользователя |
| Документ | Документы, акты, согласования |
| Согласование | Решение руководителя или смежного отдела |

## Статусы

| Статус | Смысл |
|---|---|
| Новая | Задача создана, ещё не разобрана |
| Назначена | Есть ответственный и срок |
| В работе | Исполнитель начал работу |
| Ожидает | Блокер: запчасти, доступ, согласование, данные |
| На проверке | Исполнитель закончил, нужен контроль |
| Закрыта | Работы приняты |
| Отменена | Задача неактуальна |

## Минимальное правило контроля

Каждая активная задача должна иметь ответственного и срок. Задачи без срока — управленческий туман, а в тумане обычно живут просрочки.
